# MTProto Manager

Удобный Bash-скрипт для управления прокси-серверами MTProto (Telegram) на базе Docker. Скрипт позволяет создавать, удалять и просматривать множество конфигураций прокси с поддержкой Fake TLS, автоматическим подбором портов и генерацией секретов.

## Особенности

- **Мульти-конфигурация**: Запуск нескольких независимых прокси на одном сервере.
- **Fake TLS**: Встроенная поддержка маскировки под HTTPS (домен по умолчанию `ya.ru`).
- **Умный порт**: Автоматическая проверка занятости порта 443 и выбор альтернативного (8443, 8444 и т.д.).
- **Авто-генерация**:
  - Уникальные имена контейнеров.
  - Валидные секреты для Fake TLS.
- **Чистый вывод**: Информационные сообщения без лишнего мусора, удобные для копирования.
- **Легкое управление**: Меню для добавления, удаления и просмотра активных конфигураций.

## Использование

При запуске скрипта вы увидите интерактивное меню.

### 1. Создание новой конфигурации

Выберите пункт создания нового прокси. Скрипт запросит параметры:

- **Имя конфигурации**: (Опционально) Если не ввести, будет сгенерировано автоматически (например, `mtproto-proxy-TEST-789541`).
- **Fake TLS домен**: (Опционально) По умолчанию используется `ya.ru`. Можно указать свой домен.
- **Внешний порт**: (Опционально) Если ввести занятый порт (или оставить пустым), скрипт автоматически найдет свободный.

#### Пример автоматического создания:

Если на все вопросы нажать `Enter`, скрипт сам:

1. Сгенерирует имя контейнера.
2. Проверит порт 443 (если занят → переключится на 8443).
3. Создаст секрет с hex-кодировкой домена.
4. Запустит контейнер.

---

### 2. Просмотр активных конфигураций

Выберите пункт списка активных прокси. Отображается следующая информация:

- Название конфигурации
- IP сервера
- Порт подключения
- Секрет ключа
- Домен Fake TLS
- Имя Docker-контейнера
- Прямая ссылка для подключения (`tg://proxy?...`)

---

### 3. Удаление конфигурации

Выберите конфигурацию из списка для остановки и удаления контейнера, а также очистки файлов конфигурации.

---

## Структура данных

Скрипт хранит конфигурации в директории (по умолчанию `/etc/mtproto` или текущей папке):

- `*.conf` — файлы настроек для каждой инстанции.
- Логи сохраняются внутри Docker-контейнеров.
